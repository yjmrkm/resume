# 職務経歴書

## 概要

バックエンドエンジニアとして活動しています。信条は「かっこいいコードよりも誰もが分かるコード」、「100点でお金を生まないコードよりも80点でお金を生むコード」。最近はインフラにも興味があり、少しずつ実務で知見を深めています。

**実務経験**
- WEBアプリケーションの開発
  - 7年間の実務経験
  - Go, PHP, C#
- インフラの構築
  - GCP
    - 2年間の実務経験 ※バックエンドが主
    - Redis, Cloud Tasks, IAM, etc
  - AWS
    - 3ヶ月間の実務経験
    - CDK, SQS, S3, Lambda
- デスクトップアプリケーションの開発
  - 5年間の実務経験
  - C#, VB.NET, Java
  
**得意なこと、好きなこと**
- SQLのチューニング
- 英語

## スキル(抜粋)
- チーム開発
  - 小規模(5人)かつ短期間(3ヶ月)ではありますが、プロジェクトのリーダーを務めたことがあります。作成したタスクの内容が明確で迷うことがないと言われました。
  - スクラム開発で定期的にスプリントプランニングを担当したことがあります。
  - ジュニアレベルのエンジニアのサポート役を担当していたことがあります。
  - 相互的なコードレビューの経験があります。レビューする際は柔らかい表現、レビューを受ける際は謙虚な姿勢を心掛けています。
- Go
  - 4年間の実務経験があります。
  - REST APIのスクラッチ開発を行った経験があります。Docker ComposeによるGoの開発環境整備、ライブラリの選定、アーキテクチャの検討、認証、ユニットテストなど、ひと通り担当しました。
  - APIのレスポンス速度改善を担当した経験があります。時間がかかる処理を特定し、SQLの発行回数を減らすロジック変更、Goroutineによる並行・非同期化などでパフォーマンスを改善しました。
- MySQL/PostgreSQL
  - MySQLは3年間、PostgreSQLは1年間の実務経験があります。
  - EXPLAINで実行計画を確認し、クエリをチューニングすることができます。クエリをインデックスだけで解決できるようにする、読み込みに時間がかかるTEXT型を不必要にSELECTしないなど、既存のSQL改善に従事しました。
  - PostgreSQL Anonymizerを使った動的データマスキングを導入した経験があります。
- インフラ
  - GCPやAWSにおいて、インフラ構築の実務経験があります。
  - TerraformとCDKを使った Infrastructure as Code の経験があります。
  - 運用コストを意識した設計を心掛けています。
    - ある案件でコスト最適化プロジェクトに参加していたこともあります。

**担当業務**
